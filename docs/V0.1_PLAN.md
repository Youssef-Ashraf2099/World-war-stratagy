### Phase 2: Basic Simulation (IN PROGRESS)

#### 2.1 Population Dynamics
**Goal:** Nations change population based on growth/death rates

**What to implement:**
```cpp
void DemographicPhase::execute() {
    for each province:
        newPopulation = population * (1 + birthRate - deathRate)
        population = newPopulation
    
    for each nation:
        recalculateTotalPopulation()
}
```

**Test:**
- Create province with 1M population
- Run 120 ticks (10 years)
- Verify population changes realistically

#### 2.2 Resource Production
**Goal:** Provinces produce resources based on type & infrastructure

**What to implement:**
```cpp
void EconomicPhase::execute() {
    for each province:
        production = baseYield * infraMultiplier * populationBonus
        province.resources += production
}
```

**Test:**
- Create food province with 1M pop
- Run 12 ticks
- Verify food production increases

#### 2.3 GDP Calculation
**Goal:** Nations calculate GDP from province production

**What to implement:**
```cpp
double calculateNationGDP(Nation& nation) {
    double gdp = 0;
    for each province in nation.provinces:
        gdp += province.totalProduction * resourcePrices
    return gdp;
}
```

#### 2.4 Legitimacy Calculation
**Goal:** Nations gain/lose legitimacy based on state

**What to implement:**
```cpp
double calculateLegitimacy(Nation& nation) {
    return economicGrowth 
         + militarySuccess 
         + resourceAccess
         - inequality
         - warLosses
         - tradeDependency;

```

**Test:**
- Create wealthy nation, verify high legitimacy
- Destroy economy, verify low legitimacy

### Phase 4: Determinism Validation

**Test:** Same seed → same outcome

```cpp
// Test 1: Run world A
world1.setSeed(12345);
for(120 ticks) world1.tick();
hash1 = world1.hashState();

// Test 2: Run world B (clean)
world2.setSeed(12345);
for(120 ticks) world2.tick();
hash2 = world2.hashState();

assert(hash1 == hash2, "Determinism check failed!");
```

## Code Quality Rules for v0.1

### DO
- Use strong typing (enums, not strings)
- Document assumptions with comments
- Log major decision points
- Write unit tests
- Use const correctly

### DON'T
- No global mutable state (except Logger)
- No pointer arithmetic
- No C-style arrays
- No magic numbers (use Constants.h)
- No unhandled exceptions

## Success Criteria for v0.1

✅ **Build succeeds** with no warnings
✅ **Main loop runs** for 120 ticks cleanly
✅ **3-nation test scenario** completes without errors
✅ **All values change realistically** (pop grows, GDP varies)
✅ **Simulation is deterministic** (same seed = same result)
✅ **Code compiles on Windows** (MSVC)
