## Overview

The Diplomacy System tracks bilateral relationships between nations through reputation, trade dependency, and threat alignment metrics. It evolves these relationships based on wars, shared enemies, and diplomatic actions, creating a dynamic web of international relations.

---

## Core Concepts

### Diplomatic Relation

A bidirectional relationship between two nations:

```rust
pub struct DiplomaticRelation {
    pub nation_a: NationId,
    pub nation_b: NationId,
    pub reputation: f64,           // -100 to +100
    pub trade_dependency: f64,     // 0-1 (0=independent, 1=dependent)
    pub threat_alignment: f64,     // -1 to +1 (-1=enemies, +1=allies)
    pub last_war: Option<Tick>,    // When last war occurred
    pub allied_since: Option<Tick>, // When alliance formed
    pub last_updated: Tick,        // Last modification tick
}
```

### Relationship Spectrum

```
Hostile ←────────────────────────→ Allied
   ↓                                 ↓
repute: -100                   repute: +100
threat: -1.0                   threat: +1.0
allies: Option::None           allies: Some(tick)
```

### Three Dimensions

1. **Reputation** (-100 to +100)
   - How nations view each other personally
   - Affected by: wars, proposals, shared interests
   - Decays towards neutral (0.0) each tick

2. **Trade Dependency** (0 to 1)
   - Economic interdependence
   - Affected by: active trade routes (V0.5+)
   - Currently static (set at relation creation)
   - Higher = more reason to stay peaceful

3. **Threat Alignment** (-1 to +1)
   - Strategic compatibility
   - Affected by: shared enemies, direct conflicts
   - +0.1 per shared enemy
   - -0.3 if directly at war

## Diplomacy Phase Execution

**When:** Tick phase 9 (after alliance phase, before stability)  
**Duration:** O(n²) where n = number of nations  
**Deterministic:** Yes

### Step 1: Update Relations from Wars

```rust
// Find all active wars
for each war in active_wars {
    modify_relation(war.aggressor, war.defender, -5.0)
    // Both sides lose reputation
}
```

**Logic:**
- War degrades bilateral reputation
- Both sides penalized equally
- Represents hardship of conflict

### Step 2: Update Threat Alignment

```rust
// For each nation pair
for nation_a, nation_b in nation_pairs {
    shared_enemies = count(enemies_a ∩ enemies_b)
    direct_conflict = (war(a,b) ? 1.0 : 0.0)
    
    threat_delta = (shared_enemies × 0.1) - (direct_conflict × 0.3)
    modify_threat_alignment(a, b, threat_delta)
}
```

**Logic:**
- Shared enemies increase alignment (bonds over common threats)
- Direct conflict decreases alignment (creates tension)
- Results in cooperation or rivalry based on geopolitics

### Step 3: Decay Reputation

```rust
REPUTATION_DECAY = 0.5 per tick

for each relation {
    if reputation > 0:
        reputation -= REPUTATION_DECAY
    else if reputation < 0:
        reputation += REPUTATION_DECAY
}
```

**Logic:**
- Relationships naturally drift towards neutral over time
- Represents forgetting grievances and grudges
- Relations need maintenance through continued cooperation

## Core Data Structure

```rust
impl DiplomaticRelation {
    /// Score alliance compatibility (0-1 scale)
    pub fn alliance_score(&self) -> f64 {
        (self.trade_dependency × 0.3) +
        (self.threat_alignment.abs() × 0.4) +
        ((self.reputation + 100.0) / 200.0 × 0.3)
    }
    
    /// Check if relationship is friendly
    pub fn is_friendly(&self) -> bool {
        self.reputation > 25.0
    }
    
    /// Check if relationship is hostile
    pub fn is_hostile(&self) -> bool {
        self.reputation < -25.0
    }
    
    /// Modify reputation with bounds checking
    pub fn modify_reputation(&mut self, delta: f64) {
        self.reputation = (self.reputation + delta).clamp(-100.0, 100.0)
    }
}
```

## Alliance Proposal Score Formula

The core formula determining when AI proposes alliances:

```
alliance_score = (0.3 × trade_dependency) 
               + (0.4 × |threat_alignment|) 
               + (0.3 × (reputation + 100) / 200)

Range: 0.0 (incompatible) to 1.0 (perfect allies)
Proposal Threshold: score > 0.6
```

### Formula Breakdown

**Component 1: Trade Dependency (0.3 weight)**
```
Max impact: 0.3 (when trade = 1.0)

Example:
  trade = 0.5  →  contribution = 0.15
  Higher trade = stronger economic incentive
```

**Component 2: Threat Alignment (0.4 weight - highest)**
```
Max impact: 0.4 (when |threat| = 1.0)

Example:
  threat = +1.0 (perfect allies)  →  contribution = 0.4
  threat = 0.0 (neutral)          →  contribution = 0.0
  threat = -1.0 (enemies)         →  contribution = 0.4 (still works!)
  
Note: Uses absolute value, so even enemy states can matter if
they're mutual enemies of same third party
```

**Component 3: Reputation (0.3 weight)**
```
Max impact: 0.3 (when reputation = 100)
Min impact: 0.0 (when reputation = -100)

Formula: (reputation + 100) / 200
  reputation = +100  →  (200/200) × 0.3 = 0.3
  reputation = 0     →  (100/200) × 0.3 = 0.15
  reputation = -100  →  (0/200) × 0.3 = 0.0
  
Lower reputation reduces alliance viability
```

### Score Interpretation

| Score | Interpretation | AI Action |
|-------|-----------------|-----------|
| 0.0-0.3 | Incompatible | Never propose |
| 0.3-0.6 | Possible but weak | Never propose |
| 0.6-0.75 | Strong basis for alliance | Propose |
| 0.75-0.9 | Very strong | Propose immediately |
| 0.9-1.0 | Perfect match | Propose immediately |

### Example Calculation

**Scenario: Nation A evaluating Nation B**
```
trade_dependency = 0.4  (moderate trade)
threat_alignment = 0.7  (share some enemies)
reputation = 35.0       (friendly but not close)

Score = (0.3 × 0.4) + (0.4 × |0.7|) + (0.3 × (35+100)/200)
       = 0.12 + 0.28 + 0.20
       = 0.60

Result: Score equals threshold! Would depend on rounding
        and proposal at this exact boundary.
```

## Integration with Other Systems

### War Effects

**When war declared:**
```
Reputation change:
  t=T: aggressor & defender reputation -= 5.0
  t=T+1: reputation -= 5.0 (continues each tick)
  ...

Threat alignment changes:
  Other nations see the combatants
  Those opposing defender: threat increases
  Those supporting aggressor: threat increases
```

**After war ends:**
```
Last_war timestamp updated
Reputation continues decaying towards neutral
Threat alignment stabilizes based on new alignments
```

### Alliance Integration

**When proposal made:**
```
✓ If accepted:
  → allied_since = current_tick
  → Reputation receives +5.0 boost
  → Threat alignment locks in
  
✗ If rejected:
  → Reputation -= 2.0 (diplomatic snub)
  → Threat alignment slightly decreases
```

**Within alliance:**
```
Members of same alliance:
  → Threat alignment boosted by +0.05/tick
  → Reputation gradually increases (+1.0/tick)
  → Trade dependency increases
```

### Economic System (V0.5+)

**Planned integration:**
```
Trade routes increase trade_dependency
Trade_dependency affects alliance_score
Higher dependency = more alliance pressure
Breaking alliance = trade disruption penalty
```

## Reputation Decay Mechanics

### Why Reputation Decays

Real-world diplomacy shows that:
- Nations forgive past grievances
- Alliances weaken without reinforcement
- Time heals old wounds

### The Decay Formula

```
Decay constant: 0.5 per tick

If reputation > 0 (positive):
    new_reputation = reputation - 0.5
    
If reputation < 0 (negative):
    new_reputation = reputation + 0.5
    
If reputation ≈ 0 (neutral):
    new_reputation = 0.0 (stays neutral)
```

### Decay Timeline

**Starting from +100 (perfect allies):**
```
Tick 0:   +100.0
Tick 100: +50.0   (halfway forgotten)
Tick 200: 0.0     (neutral)
Tick 300: -50.0   (become enemies)
Tick 400: -100.0  (perfect enemies) ← clamped to -100
```

**Time to forget a grudge:**
```
Starting from -50 (moderate dislike):
  Time = 50 / 0.5 = 100 ticks
  ≈ 400 hours (16+ game days at 4h/tick)
```

## API: Creating Relations Programmatically

### Create a Relation

```rust
use crate::core::types::DiplomaticRelation;

let mut world = World::new();
let nation_a = NationId::new();
let nation_b = NationId::new();

let relation = DiplomaticRelation {
    nation_a,
    nation_b,
    reputation: 0.0,
    trade_dependency: 0.3,
    threat_alignment: 0.1,
    last_war: None,
    allied_since: Some(current_tick),
    last_updated: current_tick,
};

world.spawn(relation);
```

### Query and Modify Relations

```rust
let mut query = world.query::<&mut DiplomaticRelation>();
for mut relation in query.iter_mut(&mut world) {
    if relation.nation_a == target_nation {
        // Improve relations
        relation.modify_reputation(10.0);
    }
}
```

### Calculate Alliance Score

```rust
let score = relation.alliance_score();

if score > 0.6 {
    println!("Should propose alliance!");
} else {
    println!("Wait for better relations");
}
```

## Common Patterns

### Pattern 1: Find All Relations for a Nation

```rust
let target = NationId::new();
let mut query = world.query::<&DiplomaticRelation>();

for relation in query.iter(&world) {
    if relation.nation_a == target {
        println!("Relation: {:?}", relation.nation_b);
        println!("  Score: {:.2}", relation.alliance_score());
    } else if relation.nation_b == target {
        println!("Relation: {:?}", relation.nation_a);
        println!("  Score: {:.2}", relation.alliance_score());
    }
}
```

### Pattern 2: Track Reputation Changes

```rust
// Store previous state
let previous_reputation = relation.reputation.clone();

// Do something that affects reputation
phase.execute(&mut world);

// Calculate changes
let reputation_change = relation.reputation - previous_reputation;
println!("Reputation changed by: {:.1}", reputation_change);
```

### Pattern 3: Find Best Allies

```rust
vec_relations
    .into_iter()
    .filter(|r| r.is_friendly())
    .max_by(|a, b| a.alliance_score().partial_cmp(&b.alliance_score()))
```

## Known Limitations

1. **Trade dependency is static** - Not connected to actual trade volume (V0.5+)
2. **No diplomatic incidents** - Spying/border violations not tracked (V0.5+)
3. **No relationship history** - Players can't review why relations changed
4. **Reputation decay is uniform** - All relations decay at same rate
5. **No special relationship categories** - Allies, enemies, neutrals only

## Performance

| Operation | Complexity | Time |
|-----------|-----------|------|
| Create relation | O(1) | < 0.1ms |
| Query all relations | O(r) | ~0.5ms per 100 relations |
| Calculate score | O(1) | < 0.01ms |
| Update phase (all) | O(n²) | ~10ms for 200 nations |

**Optimization Notes:**
- Threat alignment updates (n² in nations) is only expensive at very high nation counts
- Reputation decay (linear) has negligible cost
- Score calculation is O(1) mathematical operation
