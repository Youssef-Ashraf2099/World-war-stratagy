## Harness Overview

The 100K-tick stress test harness (`test_v0_6_100k_ticks_determinism`) validates:

1. **Determinism:** Same seed (99999) produces identical state across 3 independent runs
2. **Scale Stability:** Engine handles 100,000 ticks without panics/errors
3. **Performance:** Measures tick execution time (target: <15ms/tick average)
4. **State Consistency:** Nation and province counts match across all runs

### Test Configuration

```
Seed:                99999
Total Ticks:         100,000
Checkpoint Interval: 10,000 (10 checkpoints per run)
Runs:               3 (determinism validation)
Initial Nations:    20 (names: Nation_0 to Nation_19)
```

---

## How It Works

### Run 1: Baseline Execution

```
WorldState::new(99999) → Spawn 20 nations → Execute 100,000 ticks
↓
Checkpoint every 10K ticks: (tick_num, nation_count, province_count)
↓
Measure elapsed time for performance analysis
```

### Run 2: Determinism Check

```
WorldState::new(99999) → Spawn 20 nations (identical) → Execute 100,000 ticks
↓
Compare all checkpoints with Run 1:
  - Nation count must match exactly
  - Province count must match exactly
↓
Verify identical: assert_eq!(checkpoint_1[i], checkpoint_2[i])
```

### Run 3: Final Validation

```
Same as Run 2 - verify uniformity across 3 independent executions
↓
Final summary: Average ms/tick across all 3 runs
```

## Checkpoint Structure

Each checkpoint (Run 1, 2, 3) records:

```rust
Vec<(
    u64,    // tick number (10000, 20000, 30000, ..., 100000)
    usize,  // nation count (expected: 20, constant)
    usize,  // province count (may vary per run phase)

```
Checkpoint 1: tick=10000, nations=20, provinces=50
Checkpoint 2: tick=20000, nations=20, provinces=50

## Determinism Mechanism

The engine uses **DeterministicRng** (seeded) for all random decisions:

```rust
// In WorldState::new(seed: u64)
world.insert_resource(DeterministicRng::new(seed));

// All subsystems seed themselves from this RNG
// → Same seed → Same random sequence → Same outcomes
```

**Affected Subsystems:**

- AI decision-making (attacks, peace, diplomacy)
- War outcomes (casualty calculations)
- Event generation (stability, ecological, political)
- Economic variations (resource gains/losses)
- Intervention mechanics (success/failure probability)
- Refugee crisis population calculations
- Protectorate formation probability

## Performance Baselines

### Expected Tick Time

- **Target:** < 15ms/tick average
- **Estimated 100K run:** 40-60 seconds
- **3 runs total:** 120-180 seconds

### Memory Profile

- **World state:** ~50-100 MB (20 nations, 50 provinces)
- **Pipeline:** Constant memory (no accumulation between ticks)
- **Checkpoints:** Minimal (10 tuples per run)

### Bottleneck Analysis (If Needed)

If performance exceeds target:

1. Check AIPhase (typically heaviest CPU cost)
2. Check WarsPhase (casualty calculations)
3. Check EventPhase (event generation)
4. Profile with `cargo flamegraph`

---

## Success Criteria

✅ **PASS** = All of:

1. Compilation succeeds without new errors
2. All 3 runs complete without panics
3. nation_count constant = 20 across all checkpoints/runs
4. province_count identical at each checkpoint across all runs
5. Performance >= 0.1 ms/tick and <= 100ms/tick
6. No NaN or infinity in world state

❌ **FAIL** = Any of:

1. Test panics during execution
2. Checkpoint counts diverge between runs
3. Execution time > 30 minutes (100K ticks)
4. Compilation errors introduced
