## Overview

V0.6 Week 4 implements **External Intervention** mechanics, allowing neighboring nations to support factions during civil wars. This adds a diplomatic dimension to internal conflicts and creates complex multi-nation interactions.

### Key Features

1. **Intervention Detection** - Neighboring nations automatically discover civil wars
2. **Faction Support** - Nations choose which faction to support based on relations
3. **Military Aid Transfer** - Armies transferred from intervener to supported faction
4. **Resource Aid** - Food, iron, and resources provided to factions
5. **Reputation System** - Interveners gain/lose reputation based on faction outcomes
6. **Refugee Crisis** - Defeated factions produce refugees that must be integrated
7. **Protectorate Formation** - Winning factions can form lasting alliances with supporters

## System Mechanics

### 1. Intervention Detection

**Trigger:** Factionalized component added to parent nation

**Process:**

```
For each new civil war:
  1. Find all neighboring nations (adjacent provinces)
  2. For each neighbor:
     - Check diplomatic relations (reputation > -10 to consider)
     - 50% random chance to intervene
     - If intervening: select which faction to support
```

**Neighbor Detection:**

- Scans all provinces for ownership
- Identifies nations adjacent to faction provinces
- Excludes parent nation and other factions

### 2. Faction Selection

**Algorithm:**

1. Gather all factions from civil war
2. Prefer largest/strongest faction (by province count)
3. Randomize selection with deterministic RNG
4. Return selected faction NationId

**Factors Considered:**

- Faction military strength (army count)
- Faction territory (province count)
- Diplomatic ties to faction members (future phases)

### 3. Military Aid Transfer

**Amount Transferred:**

- 1-3 armies per intervention (random)
- Transferred from intervener's existing armies
- No new armies created

**Process:**

```
1. Find up to N armies owned by intervener
2. Shuffle deterministically
3. Transfer ownership to supported faction
4. Armies retain training, morale, equipment
```

**Effect:**

- Intervener loses combat power
- Supported faction gains immediate strength
- Can decisively affect war outcome

### 4. Resource Aid Transfer

**Resources Provided per Intervention:**

```
Food:        100
Iron:         50
Oil:          25
Rare Earths:  10
Water:        75
Trade Ports:   0
```

**Application:**

- Added to faction's provincial reserves
- Helps fund civil war operations
- Improves economic stability

### 5. Reputation System

**Success Bonus (+15 reputation):**

- Applied when supported faction **wins** civil war
- Creates lasting bond with faction leader
- Can lead to protectorate relationships

**Failure Penalty (-10 reputation, refugee absorption):**

- Applied when supported faction **loses**
- Generates 100k-600k refugees
- Intervener must absorb population influx
- Economic growth penalty for integration

### 6. Refugee Integration

**Mechanics (Phase 2 iteration):**

- Failed-faction members flee to intervener
- Random population pool (100k-600k per faction member)
- Temporary morale/stability penalty (-2 per 100k refugees)
- Long-term demographic growth (absorbed over 500 ticks)

**Implementation:**

- Flag added to intervener nation
- Demographics phase applies integration penalty
- Population slowly absorbs over time

### 7. Protectorate Formation

**Future Implementation (V0.6c):**

- Winning factions that received superior aid form alliance with intervener
- Protectorate alliance has:
  - Automatic military support
  - Trade agreements
  - Diplomatic protection
- 50% chance per winning faction per supporter

## Determinism & Reproducibility

**Key Design Principles:**

1. **Seeded RNG** - All random decisions use `DeterministicRng`
2. **Intervention Selection** - Deterministic given same world state + seed
3. **Army Shuffling** - Deterministic army selection via Fisher-Yates with DeterministicRng
4. **Neighbor Detection** - Deterministic query-based (no randomness)
5. **Repeatable Outcomes** - Identical seed = identical interventions

**Testing:**

- Multiple runs with same seed produce identical interventions
- Different seeds produce different but plausible interventions
- Neighboring nation set remains identical across runs

- ActiveIntervenor tracking: ~50 bytes per intervener
- Typical: 5-15 interventions per 100-nation world

## Usage Example

### Automatic Intervention (In-Engine)

```rust
// Initialization
let mut world = WorldState::new(42);
let mut pipeline = TickPipeline::new_v0_6();

// Simulate 500 ticks
// - World will run all 15 phases including intervention
pipeline.execute_many(&mut world, 500);

// Check active interventions
let interventions: Vec<&Intervention> = world
    .world
    .query::<&Intervention>()
    .iter(&world.world)
    .collect();

for intervention in interventions {
    println!(
        "Intervention: {} supports {} in {} (status: {:?})",
        intervention.intervener_nation_id,
        intervention.supported_faction_id,
        intervention.civil_war_parent_id,
        intervention.status
    );
}
```
