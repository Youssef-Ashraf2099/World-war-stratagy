## 🎯 PROJECT GOALS

### Primary Objectives

1. ✅ Create native desktop game application using Bevy Engine
2. ✅ Implement vector-based world map (no pixelation at any zoom level)
3. ✅ Design HUD inspired by Crusader Kings III / Europa Universalis IV
4. ✅ Add moderate VFX (explosions, movement trails, particles)
5. ✅ Build comprehensive developer tools suite (separate crate, feature-gated)
6. ✅ Maintain SOLID architecture principles
7. ✅ Achieve 60+ FPS on modern hardware (100+ provinces visible)

### Technical Requirements

- **Platform:** Windows/Linux/macOS native desktop application
- **Rendering:** 2D vector-based with Bevy's `bevy_render`
- **Map System:** Infinite zoom detail (0.5x to 10x+ without pixelation)
- **Assets:** Local loading from `assets/` folder (flags, icons, music, sounds)
- **API Bridge:** HTTP client to existing `alalamien-api` (REST queries)
- **Dev Tools:** Separate crate `alalamien-dev-tools`, feature-gated (`--features dev-tools`)

## 🎯 KEY TECHNICAL DECISIONS

| Decision                     | Rationale                             | Alternative Considered                      |
| ---------------------------- | ------------------------------------- | ------------------------------------------- |
| **Bevy Engine**              | Native Rust, ECS-based, excellent 2D  | Godot (would require GDScript bridge)       |
| **Vector Map**               | Infinite zoom, no pre-rendered assets | Tilemap (requires multiple texture atlases) |
| **REST API Bridge**          | Decoupled UI from engine              | Direct ECS access (tighter coupling)        |
| **Separate Dev-Tools Crate** | Clean separation, reusable            | Embedded in UI (harder to maintain)         |
| **Moderate VFX**             | 60 FPS target, balanced appeal        | High-end particles (would drop to 30 FPS)   |
