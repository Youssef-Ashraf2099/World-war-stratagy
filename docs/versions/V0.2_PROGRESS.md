### 2. Trade Subsystem (`subsystems/trade.rs`)

**Features:**

- Automatic resource redistribution within nations
- Provinces with food deficit pull from neighboring surplus provinces
- Uses `ProvinceGraph` for adjacency lookups
- Multi-pass diffusion algorithm (5 passes per tick)
- Only trades between provinces of same nation

**Algorithm:**

```rust
For 5 iterations:
  For each province with food < 0:
    Find neighbors with food > 0 (same nation only)
    Transfer food to balance deficit
```

### 3. Logistics Subsystem (`subsystems/logistics.rs`)

**Features:**

- Supply line calculation from capital
- Attrition mechanics based on distance
- Forced march penalties
- Occupation support

**Key Mechanics:**

- Armies consume resources based on distance from capital
- Supply efficiency degrades with distance
- Forced march increases attrition rate

### 4. Tick Pipeline Ordering (FIXED)

**V0.2 Pipeline Order (per ROADMAP.md):**

```rust
1. EconomicPhase      // Resource production
2. TradePhase         // Resource distribution
3. LogisticsPhase     // Supply lines & attrition
4. DemographicPhase   // Population changes
```

**Critical Fix:**
The previous order had demographics before economics, which caused:

- Population changes affecting production before resources were generated
- Non-deterministic behavior due to phase ordering

**Implementation:**

```rust
// In core/tick.rs

    Self { phases }
}
```

### 6. Stability Subsystem (`subsystems/stability.rs`) 🆕

**Features:**

- Legitimacy degradation from external and internal pressures
- Neighbor threat assessment
- Multi-front war penalties
- Resource shortage impacts
- Progressive unrest escalation: Protests → Rebels → Civil War

**Threat Calculation:**

```rust
// Calculate legitimacy loss per tick
legitimacy_loss =
    (hostile_neighbor_count * 0.5) +
    (active_war_fronts * 2.0) +
    (resource_deficit * 1.0)
```

**Escalation Thresholds:**

| Legitimacy | State         | Effects                                                                     |
| ---------- | ------------- | --------------------------------------------------------------------------- |
| < 35       | **Protests**  | ProtestEvent spawned                                                        |
| < 25       | **Rebels**    | RebelMovement spawned                                                       |
| < 15       | **Civil War** | CivilWar component, population casualties (0.1%/tick), resource destruction |

**Event Markers:**

```rust

}

    pub nation_id: NationId,

```

**Integration with Tick Pipeline:**

```rust
// Updated V0.2 pipeline order:

3. LogisticsPhase     // Supply lines & attrition
4. StabilityPhase     // Legitimacy & civil unrest 🆕
```

**Why This Order:**

- Stability runs _before_ demographics so that civil war casualties are applied before growth calculations

## 🔧 Border Data Integration

### Problem

The `ProvinceGraph` adjacency list is empty. Without border data:

- Trade cannot flow between provinces
- Logistics cannot calculate supply routes
- Blockades have no effect
- Demographics cannot simulate migration

### Solution: Natural Earth Border Extraction

**Step 1: Extract Borders**

```bash
# Requires: pip install pyshp
python scripts/extract_borders.py
```

**Output:** `src/game/scenarios/borders.json`

```json
{
  "borders": [
    {"country_a": "Canada", "country_b": "United States"},
    {"country_a": "France", "country_b": "Germany"},
    ...
  ],
  "count": 150
}
```

**Step 2: Generate Rust Border Module**

```bash
python scripts/load_borders.py
```

**Output:** `crates/alalamien-engine/src/game/borders.rs`

```rust

    vec![
        ("Canada", "United States"),
        ("France", "Germany"),
        ...
    ]
}

    // Returns bidirectional adjacency map

```

**Step 3: Integrate with WorldState**

Add to `crates/alalamien-engine/src/game/mod.rs`:

```rust
pub mod borders;
```

Add to `WorldState::from_geodata()` after spawning nations:

```rust
use crate::game::borders;

// Load border data
let border_map = borders::build_nation_adjacency_map();

// When spawning provinces for each nation, use border_map
// to call province_graph.add_border(province_a, province_b)
```

**Why This Matters:**

- **Trade:** TradePhase uses `graph.get_neighbors()` to find provinces that can exchange resources
- **Logistics:** Supply lines follow graph connectivity
- **Blockades:** Cutting edges in the graph simulates blockades
- **Demographics:** Migration can flow along borders
- **Determinism:** Pre-computed borders ensure reproducible results

## 🎓 Learning Notes for Future AI Agents

**Critical Insights:**

1. **Phase Order Matters:** The tick pipeline order is not arbitrary. Economic production must happen before trade, trade before logistics, logistics before stability, stability before demographics. This ensures resources exist before being distributed, and stability effects happen before population calculations.

2. **Border Data is Essential:** Without border data, the ProvinceGraph is useless. Make sure to run the extraction scripts and integrate the borders module.

3. **Determinism Requires Discipline:** Every `HashMap` iteration is non-deterministic by default. Sort keys before iterating. Use the ProvinceGraph carefully to maintain determinism.

4. **Natural Earth Data Structure:** The boundary lines shapefile has fields like `name_left` and `name_right` for the two countries sharing a border. The countries shapefile has polygon data for spatial adjacency analysis. The 110m scale dataset includes 177 countries, 51 US provinces/states, and 243 cities (236 capitals).

5. **Borrow Checker for World Manipulation:** When modifying World state and spawning entities, extract all immutable data (like nation names) _before_ performing any mutable operations. This avoids "cannot borrow as mutable while borrowed as immutable" errors.

6. **Legitimacy Feedback Loops:** The stability system creates natural feedback loops - low legitimacy causes protests which drain resources which further lowers legitimacy. This models realistic collapse scenarios but requires careful balancing.

7. **ECS Pattern:** Provinces are entities with components (Population, Resources, Infrastructure, OwnedBy). Subsystems iterate over entities with specific component combinations using queries.

**Common Pitfalls:**

- ❌ Don't add phases to the pipeline in the wrong order
- ❌ Don't iterate HashMap without sorting keys first
- ❌ Don't spawn provinces without populating the ProvinceGraph
- ❌ Don't modify resources in multiple phases without coordination
- ❌ Don't forget to test determinism after any changes
